Subversion Inconveniences

Subversion is in pre-Alpha state, and there are plenty of known bugs (see the Issues page for a complete list). Frankly, you should probably read over all the open issues. But since you're probably not going to do that, we list below some of the ones most likely to annoy you, and their workarounds. These will be fixed soon, sooner the more people are working on them (hint, hint).

  1. "svn up" fails if too much svndiff data received.

    While we speculate that the problem is with neon not being able handle certain large request results properly, we're not really sure. What we do know is that when doing an update of the working copy, you might see: 

       svn_error: #21073 : 
     The REPORT request failed (/repos/svn/trunk/doc/.)

   svn_error: #21073 : 
     neon: Could not read response body: connection was closed by server.

    Obvious fixes include updating invidual file targets (in theory, to make the responses to the update request smaller), or re-checking out your working copy.

    See Issue #472 for more details.

  2. Checkout of url ending in slash ("/") makes a lossy wc.

    This is a buglet we have seen before and just not gotten around to fixing. Please be careful (for now) to leave off the trailing slash from the URLs you use in checking out from a Subversion repository. Should be a trivial fix.

    See Issue #471 for more details.

  3. Bug in nested wc deletions.
    Ah, the joys and pains of versioning directories. If you delete a directory, please commit the whole directory, not just one of the (deleted) subtrees. Your working copy will thank you.

    See Issue #387 for more details.

  4. can't check out arbitrary revisions, only head

    Pretty self-explanatory. Yes, this is a major annoyance, and is up near the top of the list of post-M3 fixes.

    See Issue #411 for more details.
